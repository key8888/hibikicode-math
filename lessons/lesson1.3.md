# 第1章 数学ラボとは？
## 1.3 学習の進め方とAIとの付き合い方

### 学習目標
- 予習→実験→振り返りの学習サイクルを説明できる。
- 学習サイクルを簡単なデータで可視化し、学びのリズムをイメージする。
- AIを安全に使うためのルール（著作権・情報セキュリティ）を理解する。

## 説明＆コード
ラボでは「予習 → 実験 → 振り返り」の3ステップで進めていくよ。数学の式を覚えるだけじゃなく、プログラムで手を動かす「実験」が真ん中にあるのが特徴。例えば一次関数を例にすると、次の流れになる。

1. **予習**: 教科書や動画で \(y = ax + b\) の意味を確認する。傾きや切片がどう決まるかを整理。
2. **実験**: Pythonで `plot_function` を使い、パラメーターを動かしてグラフの変化を観察。
3. **振り返り**: 実験で気づいたことを書き出し、分からなかった点は AI や友だちに相談。

このリズムをグラフにすると、学習の進み具合が見やすくなる。下のコードでは、1週間で3サイクル回したときの理解度スコア（自分の主観でもOK）を、点と折れ線で表しているよ。

```python
# 学習サイクルごとの理解度を可視化
plot = new_plot(title="学習サイクルの理解度グラフ", width=550, height=380)

cycle_number = [1, 2, 3]
understanding_score = [40, 65, 80]  # 0〜100点で自己評価

plot_points(cycle_number, understanding_score, target_plot=plot, size=12,
            marker="triangle", color="#2ca02c", legend_label="理解度の記録")

# 折れ線でつないで流れを強調
plot_function(lambda x: 20 * x + 20, x_start=1, x_end=3, num=50,
              target_plot=plot, line_color="#1f77b4", legend_label="ざっくり傾向")
```

折れ線の式 \(y = 20x + 20\) はあくまで目安だけど、サイクルを回すほど理解度が上がるイメージがつかめるよ。自分でデータを入れ替えて、実際の成長を記録してみよう。

AIと付き合うときは、次のルールを意識しよう。

- **著作権**: 他人の文章やコードをそのまま貼り付けると、著作権（作品を作った人がもつ権利）を侵害する場合がある。引用するなら「どこから取ったか」をはっきり書く。
- **情報セキュリティ**: 個人情報（名前、学校、連絡先など）はAIに送らない。安全な範囲で質問しよう。
- **自分で考える時間を確保**: AIからの答えを鵜呑みにせず、数学の式を自分で確かめる。例えば、AIが提案した関数 \(y = 3x + 2\) が本当に問題に合っているか、グラフや計算で検証する。
