## 1.2 ラボで使うツールと学習環境

### 学習目標

- ヘルパー関数を使うと「少ないコードでも絵が出る」ことを知る。

## 説明＆コード
ラボでは、画面の左側（エディタ）でコードを書き、右側のキャンバスに結果が出る仕組みになっているよ。今回は「点を置く」だけにしぼって、道具の使い方に慣れよう。

1. **Pythonエディタ**: ここに数字や文字を入れるだけで、計算やグラフを作れる。
2. **Bokeh描画エリア**: `new_plot` を呼ぶと、ここに描画先のキャンバスができる。
3. **ヘルパー関数**: `plot_points` など、難しい設定を省いてくれる道具。引数（ひきすう: 関数に渡す追加情報）を埋めればすぐ動く。

### ステップ: 点を置いて座標を感じる
下のコードは、放課後の「立ち寄りスポット」を座標にしただけのシンプルな例だよ。まずは点が出ればOK。

```python
# 放課後に行く場所を x, y の点で置いてみる
plot = new_plot(title="放課後マップ", width=520, height=400)

# x は学校からの東西の距離、y は南北の距離と考えてみよう
places_x = [0, 1, 2.5, -1]
places_y = [0, 1.5, 0.5, -0.5]

plot_points(places_x, places_y, target_plot=plot, size=12,
            marker="circle", color="#d62728", legend_label="立ち寄りスポット")
```

`places_x` と `places_y` の順番がそろっていれば、点は正しい位置に置かれるよ。ここまでなら、グラフの知識がなくても大丈夫。

#### ミニ課題
- 自分の1日の移動を思い出して、4〜5か所の座標を決めて描いてみよう。`marker` を "square" や "triangle" に変えると形も変わるよ。

#### 振り返りチェックリスト
- [ ] コードを書く場所（エディタ）と、結果が出る場所（描画エリア）の違いが分かった。
- [ ] 座標のリストさえ作れば、点が描けることを確認できた。
- [ ] 引数を変えると色や形が変わることに気づいた。

#### 発展アイデア
- 同じ `places_x` を使いながら `places_y` だけ変えてみると、「道を東西にずらす」といったイメージがつかめるよ。
- `legend_label` を変えると凡例（グラフの説明）が変わる。自分の地図タイトルやカテゴリを入れてみよう。
